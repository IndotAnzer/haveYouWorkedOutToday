# 测试用例设计

## 1. 认证功能

### 1.1 注册功能

**正常场景：**
- 输入有效的用户名、密码和确认密码，注册成功
- 用户名长度在合理范围内（3-20个字符）
- 密码长度在合理范围内（6-20个字符）
- 确认密码与密码一致

**边界场景：**
- 用户名长度刚好为3个字符
- 用户名长度刚好为20个字符
- 密码长度刚好为6个字符
- 密码长度刚好为20个字符

**异常场景：**
- 用户名已存在
- 用户名为空
- 用户名长度超过20个字符
- 用户名长度少于3个字符
- 密码为空
- 密码长度超过20个字符
- 密码长度少于6个字符
- 确认密码与密码不一致

### 1.2 登录功能

**正常场景：**
- 输入正确的用户名和密码，登录成功
- 登录后跳转到首页
- 登录后本地存储token和userId

**边界场景：**
- 用户名和密码刚好达到最大长度

**异常场景：**
- 用户名不存在
- 密码错误
- 用户名为空
- 密码为空
- 用户名和密码都为空

## 2. 文章功能

### 2.1 创建文章

**正常场景：**
- 输入有效的标题和内容，创建成功
- 标题长度在合理范围内
- 内容长度在合理范围内

**边界场景：**
- 标题长度刚好为最大限制
- 内容长度刚好为最大限制

**异常场景：**
- 未登录尝试创建文章
- 标题为空
- 内容为空
- 标题长度超过限制
- 内容长度超过限制

### 2.2 获取所有文章

**正常场景：**
- 成功获取所有文章列表
- 文章列表按时间倒序排列
- 包含文章的基本信息（标题、内容、作者、时间等）

**边界场景：**
- 没有文章时返回空列表

**异常场景：**
- 未登录尝试获取文章（应该可以获取）

### 2.3 获取用户自己的文章

**正常场景：**
- 登录后成功获取自己的文章列表
- 文章列表按时间倒序排列

**边界场景：**
- 用户没有文章时返回空列表

**异常场景：**
- 未登录尝试获取自己的文章

### 2.4 获取单个文章详情

**正常场景：**
- 成功获取文章详情
- 包含文章的完整信息
- 包含评论和回复

**边界场景：**
- 文章ID不存在

**异常场景：**
- 未登录尝试获取文章详情（应该可以获取）
- 文章ID格式错误

### 2.5 删除文章

**正常场景：**
- 登录后成功删除自己的文章
- 删除后文章从列表中消失

**边界场景：**
- 文章ID不存在

**异常场景：**
- 未登录尝试删除文章
- 删除他人的文章
- 文章ID格式错误

## 5. 路由守卫

**正常场景：**
- 登录后可以访问需要认证的页面（如/my-articles）
- 未登录访问需要认证的页面会跳转到登录页

**异常场景：**
- 登录后token过期访问需要认证的页面
- 登录后手动删除token访问需要认证的页面
